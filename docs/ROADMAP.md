# 몇시간? 개발 로드맵

> 월 1회 이상 5만 원 초과 비필수재 구매를 하는 25-34 직장인을 위한 소비 기회비용 인식 서비스

---

## 서비스 개요

- **한 줄 정의**: 위시 상품을 추가 노동 시간으로 환산하여 소비의 기회비용을 직관적으로 인식하게 돕는 서비스
- **기술 스택**: Next.js (App Router) · TypeScript · Tailwind CSS · shadcn/ui · Supabase · Vercel · Mixpanel

---

## Phase 1 — 온보딩: 시급 설정 (Day 3~5)

### 목표
신규 사용자가 연봉을 입력하고 시급을 확인한 뒤 로컬에 저장

### 핵심 시나리오
> 진입 → 연봉(세전) 입력 → 시급 확인 → 홈 이동

### 작업 목록

#### UI
- [ ] 온보딩 페이지 (`/onboarding`) 레이아웃
- [ ] 연봉 입력 필드 (숫자만 허용, 단위: 만 원)
- [ ] 시급 계산 결과 확인 화면 (계산식 노출)
- [ ] "시작하기" CTA 버튼

#### 로직
- [ ] 시급 계산 함수
  ```
  시급 = (연봉 × 10,000) / (연간 근무시간)
  연간 근무시간 = 월 근무시간(209h) × 12
  ```
- [ ] 연봉·시급 로컬스토리지 저장
- [ ] 온보딩 완료 여부 판단 후 홈 리다이렉트

#### 추적
- [ ] `onboarding_salary_input` 이벤트 (Mixpanel)
- [ ] `onboarding_completed` 이벤트

### 완료 기준
- 숫자 외 입력 차단
- 시급이 로컬스토리지에 정상 저장됨
- 온보딩 완료 후 홈 화면으로 이동

## Phase 3 — 소비 번역기 화면 (Day 8~11)

### 목표
상품명·가격 입력 시 실시간으로 노동 시간 환산 결과를 보여주고 구매 결정을 유도

### 핵심 시나리오
> 화면 3: 소비 번역기 → 화면 4: 소비 확정 모달

### 작업 목록

#### UI — 소비 번역기 (`/translate`)
- [ ] 상품명 입력 필드
- [ ] 가격 입력 필드 (숫자만 허용, 단위: 원)
- [ ] 실시간 환산 결과 영역
  - "이 상품은 **N시간 N분**을 일해야 살 수 있어요"
- [ ] "살래요" 버튼 (Primary)
- [ ] "안 살래요" 버튼 (Primary)
- [ ] "다시 번역하기" 버튼 (Secondary)

#### UI — 소비 확정 모달 / 토스트
- [ ] "살래요" → 토스트: "N시간 N분을 더 일해야 해요" → 홈 이동
- [ ] "안 살래요" → 토스트: "N시간 N분을 지켜냈어요" → 홈 이동
- [ ] shadcn/ui `Toast` 컴포넌트 활용

#### 로직
- [ ] 가격 → 노동 시간 환산 함수
  ```
  노동 시간(분) = (가격 / 시급) × 60
  표시: N시간 N분 (예: 2시간 30분)
  ```
- [ ] 실시간 계산 (입력 디바운싱 300ms)
- [ ] 결정 결과 로컬스토리지 저장 (히스토리용)

#### 추적
- [ ] `translate_completed` 이벤트 (상품명, 가격, 환산 시간)
- [ ] `decision_buy` 이벤트
- [ ] `decision_skip` 이벤트
- [ ] `translate_reset` 이벤트

### 완료 기준
- 가격 입력 즉시 환산 결과 노출
- 결정 후 토스트 표시 → 홈 이동
- "다시 번역하기" 클릭 시 필드 초기화
